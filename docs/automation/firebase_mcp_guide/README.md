파트 1. 사전 준비
[① Node.js 설치]

Node.js는 AI와 Firebase 도구를 실행하게 해주는 엔진입니다.

nodejs.org 에 접속

왼쪽의 LTS (안정화 버전) 버튼 클릭 → 다운로드

설치 시 나오는 모든 Next 버튼 클릭하면 끝

[② Firebase 프로젝트 만들기]

Firebase 콘솔(console.firebase.google.com) 접속

구글 계정으로 로그인

[프로젝트 추가] 클릭 → 프로젝트 이름 입력 (예: philosophyAIEdu)

[③ 내 컴퓨터에 Firebase 연결하기]

터미널(Mac) 또는 CMD(Windows)를 열고 아래 명령어를 입력합니다.

1단계 — Firebase 도구 설치:
npm install -g firebase-tools

2단계 — Firebase 로그인:
firebase login

→ 브라우저 창이 뜨면 구글 로그인 완료
→ "Already logged in"이 나오면 이미 로그인된 상태
※ 노란색 경고(warn)는 정상입니다. 에러(error)만 아니면 OK!

파트 1.5. Antigravity에서 Firebase MCP 연결하기
================================================================

AI 에이전트인 Antigravity가 여러분의 Firebase 프로젝트를
직접 조작할 수 있도록 '손'을 달아주는 과정입니다.

Antigravity 실행: 에이전트 채팅창 우측 상단 [... (more_horiz)] 메뉴 클릭

MCP 서버 선택: 메뉴에서 [MCP 서버 (MCP Servers)] 클릭

Firebase 설치: 목록에서 **[Firebase]**를 찾아 [설치(Install)] 버튼 클릭

확인: 설치가 완료되면 AI가 이제부터 여러분의 프로젝트를 직접 관리할 수 있게 됩니다.

파트 2. AI에게 앱 만들어 달라고 하기
방금 MCP를 연결했다면, 이제 대화만으로 실제 클라우드 인프라를
구축하고 앱 코드를 짤 수 있습니다. 아래 순서대로 AI에게 말해 보세요.

────────────────────────────────────────────────────────────────
1단계. 프로젝트 연결 확인
────────────────────────────────────────────────────────────────

AI에게 할 말:
"내 Firebase 프로젝트 목록 보여줘"

→ AI가 내 계정에 연결된 프로젝트들을 확인해 줍니다.
→ 사용할 프로젝트 이름을 기억해 두세요 (예: philosophyAIEdu).

────────────────────────────────────────────────────────────────
2단계. 앱 초기화 + Google 로그인 + 데이터 저장 요청
────────────────────────────────────────────────────────────────

AI에게 할 말 (아래 프롬프트를 복사해서 사용하세요):

┌─────────────────────────────────────────────────────────┐
│                                                         │
│  내 Firebase 프로젝트 **"프로젝트명"**에                    │
│  아래 기능을 가진 '영어 단어 학습' 웹앱을 만들어줘.                      │
│                                                         │
│  1. Google 계정으로 로그인/로그아웃 기능                 │
│  2. 로그인한 사용자가 직접 데이터를 추가, 수정,          │
│     삭제할 수 있는 기능                                 │
│  3. 사용자가 추가한 데이터는 Firestore에 저장해서        │
│     다음에 로그인해도 그대로 남아 있게 해줘             │
│  4. 각 사용자는 자신의 데이터만 볼 수 있게 해줘          │
│                                                         │
│  데이터 구조는 Firestore에                              │
│  users/{userId} 경로 아래에 저장해줘.                   │
│                                                         │
└─────────────────────────────────────────────────────────┘

※ "프로젝트명" 부분을 자신의 Firebase 프로젝트 이름으로
바꿔서 입력하세요. (예: philosophyAIEdu)

→ AI가 firebase_init, firebase_create_app 등의 도구를 사용해
클라우드 설정을 마치고 파일들을 자동으로 만들어 줍니다.

────────────────────────────────────────────────────────────────
3단계. 로컬에서 테스트
────────────────────────────────────────────────────────────────

AI가 "다 만들었습니다!"라고 하면, Antigravity 터미널에서:
npm run dev

→ 브라우저에서 localhost 주소로 접속해 봅니다.
→ Google 로그인이 되는지, 데이터가 잘 남는지 확인합니다.
→ 확인 팁: Firebase 콘솔의 [Firestore Database] 메뉴에
방금 넣은 데이터가 들어왔는지 꼭 확인해 보세요!

파트 6. 승인된 도메인 추가 (Netlify 도메인 허용)

파트 7. Firestore 보안 규칙 설정 ★ 필수!
[7-1. Firestore 규칙 페이지로 이동]
Firebase 콘솔 → 왼쪽 메뉴 Firestore → 상단 [규칙] 탭

[7-2. 보안 규칙 작성하기]
로그인한 본인만 데이터를 다룰 수 있게 아래 코드를 붙여넣으세요.

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}

[7-3. 규칙 게시하기]
[게시] 버튼을 눌러야 실제 클라우드에 반영됩니다.
