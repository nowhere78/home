# 알파에이전트 저장소 — 작업 메모

## 외부 저장 위치 (이 git 저장소 밖, 사용자 로컬 PC)

이 저장소 외에도 사용자가 로컬 PC에 관련 자료를 보관하는 위치가 있다. **새 자료를 만들거나 "이런 게 있나?" 찾을 때, 사용자에게 매번 묻지 말고 먼저 아래 경로를 확인한다:**

- `E:\안티그라비티 자료`
- `D:\에이전트자료저장고`
- `C:\Users\smile\알파에이전트\Knowledge_Base\_company\00_Raw\reports` (src/automation/research_task.py가 참조하는 운영 경로)

> 클라우드/샌드박스 세션(Claude Code on the web 등)에는 위 드라이브가 마운트되어 있지 않아 직접 읽을 수 없다. 그런 세션에서는 추측하지 말고 "로컬 PC 세션이 아니라 이 경로는 못 확인했다"고 명확히 말한다. 사용자의 로컬 머신에서 실행되는 세션에서는 바로 확인하고 작업에 반영한다.

## 알려진 폴더 중복 — 새 파일 쓰기 전에 확인

이 저장소 안에서도 이름이 비슷한 폴더가 여러 군데로 갈라져 있다. 어디가 "진짜 최신"인지 확인 없이 새로 쓰면 데이터가 더 쪼개진다.

- `_company/_agents/*/memory.md` — ✅ 실제 운영 중인 에이전트 메모리 (가장 크고 최신, 예: `ceo/memory.md` 150KB+)
- `_agents/*/{goal,prompt}.md` — 페르소나·목표 템플릿. 수정하면 다음 자율 사이클부터 반영됨
- `Knowledge_Base/_company/` — 예전 스냅샷, 대부분 비어있음. 새 내용을 여기 쓰지 않는다

탐색 시작점은 [Home.md](Home.md) (옵시디언 vault로 열 때 진입점 노트).

## 작업 원칙
- 새 자료를 추가하기 전에 (1) 이 저장소 내 비슷한 폴더, (2) 로컬 PC의 `D:\`, `E:\` 저장 위치에 이미 비슷한 게 있는지 먼저 확인하고, 있으면 합치거나 참조한다 — 중복 생성 금지.
- 중복/혼선을 새로 발견하면 이 파일에 그 사실을 추가해서 다음 세션이 또 같은 걸 묻지 않게 한다.
