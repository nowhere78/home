## Purpose

This file is where the pack is allowed to **diverge**.

These are not claims about Anthropic or any leaked implementation. These are original design theories worth testing in your own projects.

## Hypothesis 1 - Memory should behave more like ecology than storage

Most systems treat memory as passive persistence.

A better model:
- memories compete
- memories decay
- memories reinforce each other
- hostile memories can contaminate a niche
- memory health can be measured

### Project implication
Track memory ecosystem health:
- contradiction pressure
- stale density
- poisoning pressure
- unresolved review backlog
- context-load burden

## Hypothesis 2 - Theory of mind should be multi-voice, not single-voice

Instead of one monolithic inference, generate multiple internal viewpoints:
- cautious interpretation
- generous interpretation
- skeptical interpretation
- strategic interpretation

Then compare them.

### Why
This reduces:
- projection
- emotional overclaim
- brittle one-shot inference

## Hypothesis 3 - Durable memory should require narrative compression plus structure

A memory is safer when it survives two tests:
1. it can be expressed as a structured claim
2. it can be described in a short natural-language rationale

If either fails, it is probably too fuzzy for durable promotion.

## Hypothesis 4 - The strongest defense against memory poisoning is not filtering alone, but provenance-weighted contradiction

Filtering catches obvious garbage.
But subtle poisoning often looks plausible.

A stronger design:
- allow candidate entry
- attach trust/provenance
- actively compare against existing memory
- downgrade or quarantine when contradiction patterns emerge

## Hypothesis 5 - Believability comes from bounded ignorance

World simulators and companions become more convincing when they do **not** act omniscient.

Design for:
- missing knowledge
- mistaken beliefs
- delayed realization
- asymmetry of information

## Hypothesis 6 - Agency quality is a balance of power and humility

High-agency systems often fail by pushing too far.
Low-agency systems fail by stalling.

A mature agent should combine:
- initiative
- boundedness
- reversibility
- self-doubt where appropriate

## Hypothesis 7 - Relationship continuity needs weather and climate

Use two layers:
- **weather** = short-term relational tone
- **climate** = long-term relationship structure

This is useful for both companion systems and political/social world simulation.

## Hypothesis 8 - The best compaction output is delta-oriented, not essay-oriented

Instead of one giant summary, produce:
- what changed
- what remains unresolved
- what grew more uncertain
- what should be forgotten soon
- what became more trusted

## Hypothesis 9 - Permissions should optimize for regret, not simply danger

Ask:
- if we allow this and it is wrong, how costly is regret?
- if we block this and it was safe, how much helpfulness is lost?

This reframes permissioning around practical harm and usability.

## Hypothesis 10 - Long-running agents need ritual checkpoints

Not mystical ritual necessarily, but structured pauses:
- restate goal
- summarize progress
- inspect risk
- inspect budget
- inspect memory proposals
- choose continue/branch/stop

### Why
This reduces drift and runaway loops.

## Hypothesis 11 - World engines need sacred facts and mortal facts

A useful split for lore-heavy systems:
- **sacred facts**: authored anchors, hard to change
- **mortal facts**: everyday state, easier to update

This stops emergent simulation from trampling core mythic or authored structure.

## Hypothesis 12 - Good subagents should have different memory metabolisms

Not every agent should remember the same way.

Examples:
- reviewer remembers risks and regressions
- patcher remembers local task state
- lore-keeper remembers canon anchors
- companion-style layer remembers relational patterns

## Hypothesis 13 - Context should be assembled like a spell circle, not dumped like a cargo cart

Meaning:
- every piece should have a reason to be present
- layers should be concentric
- central task first
- peripheral background only as needed
- hostile material kept outside the inner ring unless explicitly examined

## Hypothesis 14 - Explainability is a weapon against both bugs and manipulation

If the system can answer:
- why it believed this
- why it acted
- why it refused
- why it remembered
then both engineering and trust improve.
