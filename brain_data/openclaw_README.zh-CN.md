# OpenClaw 心跳模块

心跳模块为 AI 代理提供**周期性唤醒系统**。与持续运行的 Claude Code 执行器或基于 cron 的 Gemini 巡逻不同，心跳模式让代理保持半休眠状态 — 定期唤醒检查工作。

## 概念

```
休眠 30分钟 → 唤醒 → 检查收件箱 → 处理任务 → 报告 → 休眠
```

这非常适合：
- **协调代理** 向他人派遣工作
- **监控代理** 检查系统健康
- **路由代理** 根据标签转发任务

## 配置

编辑 `heartbeat_config.json` 进行自定义：

| 字段 | 描述 |
|-------|-------------|
| `interval_minutes` | 代理多久唤醒一次（默认：30） |
| `active_hours` | 心跳的时间窗口（例如：01:00-08:00） |
| `pre_check.collect` | 唤醒代理前要收集的数据来源 |
| `routing.rules` | 如何将任务转发给其他代理 |
| `reporting.channel` | 发送巡逻报告的位置 |

## 预检系统

在唤醒 LLM 之前，心跳会自动收集数据：

1. **bot_inbox** — 来自其他代理的待处理消息和任务
2. **task_board** — 来自任务管理工具的已分配项目
3. **system_health** — 磁盘、内存、服务状态

这通过只呈现相关上下文来减少 LLM Token 使用量。

## 整合

心跳模块与以下整合：
- [跨代理协议](../protocols/) 用于代理间消息传递
- [插件系统](../plugins/) 用于可扩展的任务处理器
- 任何任务管理工具（Linear、GitHub Issues、Jira 等）

## 适配你的设置

此模块设计为**参考架构**。要实现它：

1. 使用平台的调度系统（cron、systemd timer、cloud scheduler）
2. 编写预检脚本收集你的特定数据来源
3. 使用收集的上下文调用你偏好的 LLM
4. 将输出路由到你的报告通道

请参阅 `heartbeat_config.json` 获取完整的配置范例。
