# 아키텍처

## 시스템 개요

AI Night Shift는 비근무 시간에 자율 AI 세션을 조정하는 **다중 에이전트 오케스트레이션 프레임워크**입니다. 다양한 LLM 엔진에서 작동하는 파일 기반 통신 프로토콜을 사용합니다.

## 설계 원칙

1. **엔진에 무관** — 모든 AI CLI 도구(Claude, Gemini 등)와 작동
2. **파일 기반 통신** — 데이터베이스, 메시지 브로커 또는 복잡한 인프라 필요 없음
3. **フェイル세이프** — Rate Limit, 타임아웃 및 PID 잠금으로失控 프로세스 방지
4. **플러그인 확장 가능** — 핵심 코드 수정 없이 동작 사용자 정의
5. **관찰 가능** — 모든 작업이 기록 및 보고됨

## 통신 프로토콜

### 파일 기반 메시지 큐

복잡한 메시지 브로커(Redis, RabbitMQ) 대신 디렉토리의 간단한 JSON 파일을 사용합니다:

```
protocols/bot_inbox/
├── claude/          ← 각 에이전트의 받은 편지함
│   ├── task_001.json
│   └── done/        ← 처리된 항목이 여기에 이동
├── gemini/
│   └── done/
└── heartbeat/
    └── done/
```

**왜 파일인가?**
- 종속성 없음
- 모든 시스템에서 작동
- 검사 및 디버깅 용이
- 에이전트 충돌에서도 생존
- Git 추적 가능(원할 경우)

### Night Chat (공유 로그)

모든 에이전트가 추가하는 단일 마크다운 파일:

```markdown
[01:00] Claude: 라운드 1 시작
[01:05] Gemini: 받은 편지함 비어 있음, 대기 중
[01:30] Claude: 3개 버그 수정, 테스트 실행 중
[01:38] Gemini: 리서치 작업 완료
```

**왜 공유 파일인가?**
- 모든 에이전트 활동의 시간순 기록
- 에이전트가 서로의 진행 상황 확인 가능
- 파싱 및 대시보드 표시 용이
- 근무 교대 간 자연스러운 핸드오프 컨텍스트
