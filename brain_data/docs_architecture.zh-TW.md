# 架構

## 系統概述

AI Night Shift 是一個**多代理協調框架**，在非工作時間協調自主 AI 會話。它使用基於檔案的通訊協定，可跨不同 LLM 引擎運作。

## 設計原則

1. **引擎無關** — 適用於任何 AI CLI 工具（Claude、Gemini 等）
2. **基於檔案的通訊** — 無需資料庫、訊息代理或複雜基礎設施
3. **安全機制** — 速率限制、超時和 PID 鎖防止失控程序
4. **外掛可擴展** — 無需修改核心程式碼即可自訂行為
5. **可觀測性** — 每個動作都會被記錄和報告

## 通訊協定

### 基於檔案的訊息隊列

我們使用目錄中的簡單 JSON 檔案，而不是複雜的訊息代理（Redis、RabbitMQ）：

```
protocols/bot_inbox/
├── claude/          ← 每個代理都有收件匣
│   ├── task_001.json
│   └── done/        ← 處理後的項目移到此處
├── gemini/
│   └── done/
└── heartbeat/
    └── done/
```

**為什麼用檔案？**
- 零依賴
- 適用於任何系統
- 易於檢查和調試
- 代理崩潰後仍可恢復
- 可 Git 追蹤（如需要）

### Night Chat（共享日誌）

所有代理都會附加到的單一 Markdown 檔案：

```markdown
[01:00] Claude: 開始回合 1
[01:05] Gemini: 收件匣為空，待命中
[01:30] Claude: 修復 3 個錯誤，執行測試中
[01:38] Gemini: 研究任務完成
```

**為什麼用共享檔案？**
- 所有代理活動的時間順序記錄
- 代理可以看到彼此的進度
- 易於解析並在儀表板顯示
- 班次之間的自然交接上下文
