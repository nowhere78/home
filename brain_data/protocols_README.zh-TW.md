# 跨代理通訊協定

協定模組是多代理夜班的骨幹。它定義了**不同的 AI 代理如何通訊、共享上下文和協調工作** — 即使它們運行在不同的 LLM 引擎上。

## 架構概述

```
┌─────────────┐     night_chat.md      ┌─────────────┐
│  Claude Code │ ◄──────────────────►  │   Gemini     │
│  (開發者)   │                        │  (研究者)   │
└──────┬───────┘                        └──────┬───────┘
       │           ┌─────────────┐             │
       │           │  Heartbeat  │             │
       └──────────►│ (協調者)    │◄────────────┘
                   └──────┬───────┘
                          │
                   ┌──────▼───────┐
                   │  任務面板     │
                   │ (Linear/GH)  │
                   └──────────────┘
```

## 通訊通道

### 1. 夜聊 (`night_chat.md`)

代理發布即時狀態更新的共享 Markdown 檔案。把它想像成檔案中的團隊 Slack 頻道。

**格式：**
```
[HH:MM] 代理名稱: 訊息內容
[HH:MM] 代理名稱: 另一個更新
```

**規則：**
- 僅附加（從不覆蓋現有訊息）
- 保持訊息簡潔（1-2 行）
- 包含時間戳以進行時間順序追蹤
- 任何代理都可以閱讀；每個代理附加訊息

**用例：**
- 長時間任務期間的進度更新
- 向其他代理請求協助
- 分享發現或警告

### 2. 機器人收件匣 (`bot_inbox/`)

使用 JSON 檔案的結構化訊息隊列。每個代理都有自己的收件匣目錄。

**結構：**
```
bot_inbox/
├── claude/          ← Claude Code 代理的訊息
│   ├── task_001.json
│   ├── msg_002.json
│   └── done/        ← 已處理的訊息
├── gemini/          ← Gemini 代理的訊息
│   └── done/
└── heartbeat/       ← Heartbeat 代理的訊息
    └── done/
```

**訊息格式：**
```json
{
  "from": "claude",
  "to": "gemini",
  "type": "message",
  "task_id": "TASK-123",
  "message": "Research completed, results in /reports/",
  "ts": "2026-03-11T14:45:00Z"
}
```

**支援的類型：**
- `message` — 自由格式通訊
- `task` — 工作分配
- `result` — 任務完成報告

### 3. 通知腳本 (`notify.sh`)

用於向其他代理報告任務完成。

```bash
./notify.sh <目標代理> <任務ID> <狀態> <摘要>
```

**範例：**
```bash
./notify.sh gemini TASK-42 success "重構了 auth 模組，12 個測試通過"
```

### 4. 訊息腳本 (`msg.sh`)

用於代理之間的直接訊息。

```bash
./msg.sh <目標代理> "你的訊息內容"
```

## 協定規則

1. **僅附加** — 從不刪除或修改其他代理的訊息
2. **處理和歸檔** — 將處理的收件匣項目移到 `done/`
3. **結構化優於自由格式** — 對於可操作的項目首選 JSON 收件匣，夜聊用於 FYI 更新
4. **冪等處理** — 代理應該優雅地處理重複訊息
5. **超時感知** — 如果訊息在 2 個心跳週期內未處理，則升級

## 任務面板整合

協定與任何任務管理系統配合使用。範例：

| 系統 | 整合 |
|--------|-------------|
| Linear | `linear_tool.py create/update/comment` |
| GitHub Issues | `gh issue create/edit/comment` |
| Jira | Jira CLI 或 API |
| 純檔案 | 帶有勾選格式的 `tasks.md` |

關鍵原則：**每個任務分配都應該存在於面板上**，而不只是在聊天中。聊天用於協調；面板是真相來源。
