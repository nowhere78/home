# 架构

## 系统概述

AI Night Shift 是一个**多代理编排框架**，在非工作时间协调自主 AI 会话。它使用基于文件的通讯协议，可跨不同 LLM 引擎运行。

## 设计原则

1. **引擎无关** — 适用于任何 AI CLI 工具（Claude、Gemini 等）
2. **基于文件的通讯** — 无需数据库、消息代理或复杂基础设施
3. **安全机制** — 速率限制、超时和 PID 锁防止失控进程
4. **插件可扩展** — 无需修改核心代码即可自定义行为
5. **可观测性** — 每个动作都会被记录和报告

## 通讯协议

### 基于文件的消息队列

我们使用目录中的简单 JSON 文件，而不是复杂的消息代理（Redis、RabbitMQ）：

```
protocols/bot_inbox/
├── claude/          ← 每个代理都有收件箱
│   ├── task_001.json
│   └── done/        ← 处理后的项目移到此处的
├── gemini/
│   └── done/
└── heartbeat/
    └── done/
```

**为什么用文件？**
- 零依赖
- 适用于任何系统
- 易于检查和调试
- 代理崩溃后仍可恢复
- 可 Git 追踪（如需要）

### Night Chat（共享日志）

所有代理都会附加到的单一 Markdown 文件：

```markdown
[01:00] Claude: 开始回合 1
[01:05] Gemini: 收件箱为空，待命中
[01:30] Claude: 修复 3 个错误，执行测试中
[01:38] Gemini: 研究任务完成
```

**为什么用共享文件？**
- 所有代理活动的时序记录
- 代理可以看到彼此的进度
- 易于解析并在仪表板显示
- 班次之间的自然交接上下文
