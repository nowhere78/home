# OpenClaw 하트비트 모듈

하트비트 모듈은 AI 에이전트를 위한 **주기적 각성 시스템**을 제공합니다. 지속적인 Claude Code 실행기나 cron 기반 Gemini 순찰과 달리, 하트비트 모델은 에이전트를半잠眼 상태로 유지합니다 — 정기적으로 각성하여 작업을 확인합니다.

## 개념

```
30분.Sleep → 각성 → 받은 편지함 확인 → 작업 처리 → 보고 → Sleep
```

다음에 이상적입니다:
- 다른 에이전트에 작업을 보내는 **코디네이터 에이전트**
- 시스템 상태를 확인하는 **모니터링 에이전트**
- 라벨을 기반으로 작업을 전달하는 **라우터 에이전트**

## 구성

`heartbeat_config.json`을 편집하여 사용자 정의:

| 필드 | 설명 |
|-------|-------------|
| `interval_minutes` | 에이전트가 얼마나 자주 각성하는지 (기본값: 30) |
| `active_hours` | 하트비트의 시간 창 (예: 01:00-08:00) |
| `pre_check.collect` | 에이전트를 깨우기 전에 수집할 데이터 소스 |
| `routing.rules` | 작업을 다른 에이전트에 전달하는 방법 |
| `reporting.channel` | 순찰 보고서를 보내는 위치 |

## 사전 확인 시스템

LLM을 깨우기 전에 하트비트가 자동으로 데이터를 수집합니다:

1. **bot_inbox** — 다른 에이전트의 보류 중인 메시지와 작업
2. **task_board** — 작업 관리 도구의 할당된 항목
3. **system_health** — 디스크, 메모리, 서비스 상태

관련 컨텍스트만 제시하여 LLM 토큰 사용량을 줄입니다.

## 통합

하트비트 모듈은 다음과 통합됩니다:
- 에이전트 간 메시징을 위한 [跨에이전트 프로토콜](../protocols/)
- 확장 가능한 작업 처리를 위한 [플러그인 시스템](../plugins/)
- 모든 작업 관리 도구 (Linear, GitHub Issues, Jira 등)

## 설정에 맞게 조정

이 모듈은 **참조 아키텍처**로 설계되었습니다. 구현하려면:

1. 플랫폼의 스케줄링 시스템 사용 (cron, systemd timer, cloud scheduler)
2. 특정 데이터 소스를 수집하는 사전 작성
3. 확인 스크립트 수집된 컨텍스트로 선호하는 LLM 호출
4. 출력을 보고 채널로 라우팅

전체 구성 예제는 `heartbeat_config.json`을 참조하세요.
