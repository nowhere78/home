# 🛠️ 팩트체크 및 분량 보완 (Kaggle QA 시스템) 구축 계획

회원님의 예리한 지적대로, 생성된 글들을 자체 검수한 결과 **심각한 오류(할루시네이션)**와 **분량 부족**이 발견되었습니다. 
*(예: 식물인 '타트체리'를 '말레이시아 돼지에서 추출했다'고 엉뚱하게 소설을 쓰거나, 분량이 1800자 내외로 짧게 끝난 현상)*

이러한 문제는 8B(경량화) 사이즈인 Llama 3.1 모델이 가진 의학 지식의 한계 때문에 발생합니다. 
회원님의 지시("올라마를 통해 하되 부족한 부분은 네가 보완해라")에 따라, **제가 Llama의 지능을 극한으로 끌어올리고 팩트를 통제하는 '특수 프롬프트와 논리 구조'를 스크립트에 보완(Supplement)**하여, 기존 글들을 완벽하게 재건축하는 계획을 세웠습니다.

## 1. 문제 해결 접근법 (Gemini의 보완책)
Llama 3.1에게 단순히 "다시 써!"라고 하면 똑같은 헛소리를 반복합니다. 따라서 다음과 같은 장치를 Kaggle 코드에 탑재합니다.
*   **강제 팩트체크 로직**: 타트체리, 여주, 새싹보리 등은 '식물/천연물'임을 시스템 프롬프트에 강제로 주입하여 동물이나 엉뚱한 추출물로 창조하는 것을 원천 차단합니다.
*   **목차(구조) 강제 할당**: 분량을 3,000자 이상으로 억지로 늘리기 위해, Llama에게 다음과 같은 5개의 목차를 무조건 작성하도록 강제합니다.
    1. 정확한 정의와 원료의 기원 (식물성/미네랄 등 명시)
    2. 체내 작용 메커니즘 (해부학적/생리학적 근거)
    3. 구체적인 효능 3가지
    4. 치명적인 부작용과 복용 금기 대상 (임산부, 특정 약물 복용자)
    5. 올바른 섭취 방법과 하루 권장량
*   **웹 검색(RAG) 도입 (선택적)**: 필요하다면 위키피디아 등에서 실제 정보를 긁어와서 Llama에게 읽힌 뒤 요약하게 만들어 소설 쓰는 것을 방지합니다. (이번 코드에서는 프롬프트 엔지니어링으로 먼저 통제합니다.)

## 2. Kaggle QA (품질 관리) 코드 자동화
새로 작성해 드릴 Kaggle 코드는 다음과 같이 작동합니다.
1. 워드프레스에 접속해 오늘 작성된 52개의 예약 글을 다시 모두 불러옵니다.
2. 글자 수가 너무 적거나(3000자 미만), 내용이 엉망인 글들을 찾아냅니다.
3. 제가 보완해 둔 **'초정밀 의학 검수용 프롬프트'**를 Llama 3.1에 주입하여 기존 내용을 완전히 새롭고 길게, 그리고 '정확하게' 다시 작성(Rewrite)합니다.
4. 이미 썸네일은 완벽하게 들어가 있으므로, **글의 내용(Text)만 덮어쓰기(Update)** 하여 품질을 극상으로 끌어올립니다.
