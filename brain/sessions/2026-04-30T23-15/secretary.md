## 📑 Omega Risk Index (ORI) 개발 요구사항 정의서 (MVP 스펙)

CEO님의 지시에 따라, Developer가 구현해야 할 '오메가 리스크 지수'의 핵심 로직 및 구조를 최종적으로 확정하고 문서화했습니다. 이 문서는 개발 단계에서 요구되는 모든 변수, 가중치, 판정 기준을 포함하며, API 구현의 근거 자료로 활용되어야 합니다.

---

### 1. 📊 핵심 입력 변수 정의 및 가중치 (Input Variables & Weights)

ORI는 5가지 핵심 변수에 대한 1~5점 척도(5점이 최고 점수) 점수 입력을 받습니다. 각 변수별 가중치는 해당 변수가 전체 리스크 지수에 미치는 상대적 중요도를 나타냅니다.

| 변수명 (Variable) | 정의 (Definition) | 가중치 (Weight) | 비고 |
| :--- | :--- | :--- | :--- |
| **재정안정성** | 자금 흐름의 안정성, 현금 창출 구조의 견고함 | **0.30** | (가장 중요) |
| **시장적응력** | 시장 변화에 대한 민첩한 반응 및 사업 모델의 확장성 | **0.25** | |
| **경험축적도** | 유사 성공/실패 경험의 누적량 및 체계화 정도 | **0.20** | |
| **네트워크활성화** | 핵심 파트너, 멘토, 인맥 등 외부 자원의 연결성 | **0.15** | |
| **자기계발의지** | 새로운 지식 습득 및 역량 강화에 대한 지속적인 노력 | **0.10** | |
| **총합** | | **1.00** | |

### 2. ⚠️ 위험 등급 산정 기준 (Risk Grading Thresholds)

총 리스크 지수는 0.0부터 1.0 사이의 값을 가지며, 다음 임계값(Threshold)에 따라 3가지 등급으로 분류됩니다.

| 리스크 등급 (Grade) | 점수 범위 (Score Range) | 해석 및 행동 지침 (Action) |
| :--- | :--- | :--- |
| **Low (낮음)** | $0.0 \le \text{Score} \le 0.35$ | **[기회/Opportunity]** 리스크가 낮아 성공 가능성이 높음. 다음 단계의 자원 투입 및 확장 계획 수립에 집중. |
| **Medium (보통)** | $0.35 < \text{Score} \le 0.70$ | **[주의/Caution]** 중간 수준의 리스크. 취약 변수를 식별하고, 해당 영역에 대한 보강 계획(Action Plan) 수립이 필수. |
| **High (높음)** | $0.70 < \text{Score} \le 1.0$ | **[경고/Warning]** 리스크가 매우 높음. 즉각적인 방향 수정(Pivot) 또는 리소스 확보를 위한 전면 재점검(Audit)이 필요함. |

### 3. 🗺️ 사용자 플로우차트 (User Flowchart - MVP)

사용자가 데이터를 입력하고 결과물을 받는 과정은 다음의 4단계로 정의됩니다.

1.  **[Input]** 사용자 (혹은 내부 데이터 수집 에이전트)가 5대 변수별 점수(1~5점)를 시스템에 제출한다.
2.  **[Process]** 백엔드 로직이 다음 공식을 사용하여 가중치 기반 총점을 계산한다.
    $$\text{ORI} = \sum_{i=1}^{5} (\text{Score}_i \times \text{Weight}_i)$$
3.  **[Output]** 계산된 총점(ORI)과 이를 기반으로 산정된 위험 등급(Low/Medium/High)을 반환한다.
4.  **[Presentation]** 최종 결과와 함께, **가장 낮은 점수를 받은 1~2개 변수**를 식별하여 취약점을 시각적으로 강조한다.

### 4. 💻 개발자 구현 요청 상세 명세 (Developer Deliverables)

| 항목 | 상세 요구사항 |
| :--- | :--- |
| **API 엔드포인트** | `POST /api/v1/omega-risk` (예시) |
| **요청 (Request Body)** | JSON 형식. 5대 변수 키와 점수(1~5)를 포함해야 함. <br>예시: `{"재정안정성": 4, "시장적응력": 3, "경험축적도": 5, "네트워크활성화": 2, "자기계발의지": 4}` |
| **응답 (Response Body)** | JSON 형식. 다음 3가지 요소를 반드시 포함해야 함. <br>1. `index_score`: 계산된 총점 (float, 소수점 2자리). <br>2. `risk_grade`: 판정된 등급 (string: Low/Medium/High). <br>3. `details`: 변수별 점수 및 가중치 기반의 세부 계산 점수 딕셔너리. |
| **예외 처리** | 입력 변수가 5가지가 아니거나, 점수 범위(1~5)를 벗어날 경우, 명확한 에러 메시지를 반환해야 함. |
