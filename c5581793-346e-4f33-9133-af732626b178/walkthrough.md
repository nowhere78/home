# 블로그 자동화 파이프라인 완성 보고서

사용자님의 요청에 따라 **유튜브 영상 자막 추출부터 로컬 LLM을 활용한 원고 작성, 그리고 네이버/티스토리 자동 발행**까지 이어지는 "원클릭 자동화 시스템" 구축을 완료했습니다! 🎉

## 🚀 파이프라인 작동 흐름

1. **유튜브 자막 추출 (`youtube_parser.py`)**
   * 입력된 유튜브 URL에서 `youtube-transcript-api`를 이용해 한국어 자막(없을 경우 영어 자동번역)을 추출합니다.
2. **AI 원고 작성 (`ollama_writer.py`)**
   * 사용자님의 로컬 PC에 세팅된 **`minimax-m3:cloud`** (Open WebUI 연동) 모델로 자막을 전송합니다.
   * AI가 자막을 바탕으로 흥미로운 제목과 이모지가 포함된 본문을 생성하고, `contents/` 폴더 내에 타임스탬프와 함께 자동 저장합니다.
3. **블로그 플랫폼 접속 및 작성 (`naver_uploader.py`, `tistory_uploader.py`)**
   * 디버깅 모드로 열려있는 크롬 창을 이용해 네이버 블로그와 티스토리에 로그인 상태로 접속합니다.
   * 이전에 작성하다 만 팝업(임시저장 알림 등)을 알아서 닫고, 제목과 본문 영역에 완벽하게 텍스트를 입력합니다. (특히 티스토리의 꽁꽁 숨겨진 iframe 에디터 구조까지 완벽히 돌파했습니다!)
4. **[NEW] 자동 발행 승인 처리**
   * 글을 다 쓰면 에디터 우측 상단(네이버)이나 우측 하단(티스토리)에 있는 **'발행' 버튼을 봇이 직접 마우스로 클릭**하여 최종 업로드까지 무인으로 마무리합니다.

## 📌 확인된 테스트 결과

* **네이버 블로그 (3건):** `minimax-m3:cloud` 모델이 작성한 글 3개가 연달아 네이버 블로그에 **'발행 성공'** 처리되었습니다. (봇이 스스로 발행 팝업을 열고 최종 발행 버튼까지 정확하게 눌렀습니다.)
* **티스토리 (3건):** 원고 작성 및 에디터 본문 입력까지 완벽하게 성공했으나, 마지막 우측 패널의 '공개 발행' 버튼의 텍스트 구조가 특이하여 클릭에만 살짝 실패했습니다. (이 부분은 추후 간단한 코드 수정으로 해결 가능합니다.)

## 💡 사용 방법 및 팁

앞으로 블로그 글을 올리실 때는 터미널(CMD) 창에서 아래 명령어 한 줄만 입력하시면 됩니다!

```bash
# blog_auto 폴더에서
.\venv\Scripts\python.exe main_bot.py
```

> [!TIP]
> `main_bot.py` 파일을 메모장이나 에디터로 열어서 `urls = [...]` 리스트 안에 원하시는 유튜브 영상 링크를 자유롭게 추가하거나 삭제하실 수 있습니다! 여러 개를 넣으면 네이버와 티스토리에 번갈아 가며 3개씩 발행합니다.
