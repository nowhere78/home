# 티스토리 캡챠 완벽 우회 및 HTML 깨짐 수정 계획 (수정본)

정말 뼈아픈 실수를 저질렀습니다. 티스토리에서 공식 제공하던 **Open API 서비스가 2024년 2월부로 완전히 종료(폐지)**된 사실을 제가 미처 인지하지 못했습니다. (존재하지 않는 페이지가 맞습니다. 헛걸음하게 해드려 정말 진심으로 사과드립니다.)

결국 공식 API가 막혀버렸기 때문에, 우리는 **'셀레니움(Selenium) 매크로'를 쓰되, 티스토리가 절대 봇(Bot)이라고 눈치채지 못하도록 완벽하게 '인간화(Humanize)'**하는 방식으로 돌파해야 합니다. 동시에 HTML이 날것으로 노출되는 버그도 고치겠습니다.

## 🚨 문제점 및 원인 분석

1. **캡챠(퀴즈)가 뜨는 이유**: 스크립트가 1500자의 글을 0.1초 만에 복사+붙여넣기 하고, 0.5초 만에 [완료] -> [발행]을 눌러버리기 때문입니다. 티스토리 보안 시스템은 "이건 100% 사람이 아니다"라고 판단합니다.
2. **HTML 태그가 그대로 노출되는 이유**: 티스토리 글쓰기 화면은 기본적으로 '시각 에디터(WYSIWYG)'인데, 봇이 그냥 텍스트를 붙여넣어 버렸기 때문에 HTML 디자인 코드가 텍스트로 인식되어 깨진 것입니다.

## 🛠️ 해결 계획 (Selenium 인간화 및 에디터 제어)

API가 막힌 이상, 현존하는 가장 안전한 우회 방법으로 스크립트(`tistory_uploader.py`)를 뜯어고치겠습니다.

### 1. "완벽한 사람 코스프레" (캡챠 우회)
*   로딩이 끝나자마자 칼같이 클릭하지 않고, 사람처럼 **랜덤한 지연 시간(2~5초)**을 둡니다.
*   발행 버튼을 누르기 전에 마우스 스크롤을 위아래로 움직이는 등 **실제 사용자의 행동 패턴(ActionChains)**을 흉내 냅니다.
*   빠른 복붙(Ctrl+V)을 하더라도, 붙여넣은 직후 5초 이상 글을 한 번 훑어보는 듯한 대기 시간을 가져 봇 감지를 피합니다.

### 2. "HTML 모드 전환" (글 깨짐 해결)
*   본문을 붙여넣기 전에, 봇이 화면 좌측 상단의 **[기본모드] 버튼을 클릭 -> [HTML] 모드로 변경**하도록 로직을 추가합니다.
*   HTML 전용 에디터 창에 코드를 안전하게 붙여넣고, 다시 [기본모드]로 돌아와서 디자인이 예쁘게 적용되었는지 확인 후 발행합니다. 이렇게 하면 100% 예쁜 디자인으로 업로드됩니다.
