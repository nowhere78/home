claude -r
workflow: pick a Claude Code session, import it into Codex, and jump straight into the imported Codex session.
codex -r
[90] (622,829)<clipboard-copy aria-label='Copy' class='ClipboardButton btn btn-invisible js-clipboard-copy m-2 p-0 d-flex flex-justify-center flex-items-center' role='button' />
It exists because Codex added external agent session import, but the first usable path can be hard to trigger. The startup prompt depends on the
external_migration
feature flag and the trust onboarding flow. In projects that are already trusted, you may never see the prompt. Large
~/.claude/projects
folders can also make home-wide detection slow.
If you move between Claude Code and OpenAI Codex, CODEX-R turns session migration into a picker instead of a scavenger hunt.
