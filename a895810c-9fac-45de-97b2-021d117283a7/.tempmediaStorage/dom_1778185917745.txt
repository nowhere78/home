Open Agents는 Vercel에서 백그라운드 코딩 에이전트를 구축하고 실행하기 위한 오픈 소스 참조 애플리케이션입니다. 웹 UI, 에이전트 런타임, 샌드박스 오케스트레이션 및 GitHub 통합 기능을 포함하여 노트북을 사용하지 않고도 프롬프트에서 코드 변경까지 바로 진행할 수 있도록 지원합니다.
이 저장소는 블랙박스처럼 취급하기보다는 포크하고 수정할 수 있도록 만들어졌습니다.
그것은 무엇인가
[41] (193,462)<a aria-label='고유 링크: 이것은 무엇인가' class='anchor' href='#what-it-is' id='user-content-what-it-is' />
Open Agents는 3계층 시스템입니다.
Web -> Agent workflow -> Sandbox VM
[42] (626,570)<clipboard-copy aria-label='Copy' class='ClipboardButton btn btn-invisible js-clipboard-copy m-2 p-0 d-flex flex-justify-center flex-items-center' role='button' />
웹 애플리케이션은 인증, 세션, 채팅 및 스트리밍 UI를 처리합니다.
해당 에이전트는 Vercel에서 내구성이 뛰어난 워크플로로 실행됩니다.
샌드박스는 실행 환경입니다. 여기에는 파일 시스템, 셸, Git, 개발 서버 및 미리 보기 포트가 포함됩니다.
핵심적인 아키텍처 결정: 에이전트는 샌드박스가 아니다
[43] (193,742)<a aria-label='고유 링크: 핵심 아키텍처 결정: 에이전트는 샌드박스가 아닙니다.' class='anchor' href='#the-key-architectural-decision-the-agent-is-not-the-sandbox' id='user-content-the-key-architectural-decision-the-agent-is-not-the-sandbox' />
에이전트는 가상 머신 내부에서 실행되지 않습니다. 샌드박스 외부에서 실행되며 파일 읽기, 편집, 검색 및 셸 명령과 같은 도구를 통해 가상 머신과 상호 작용합니다.
그 분리가 바로 이 프로젝트의 핵심입니다.
에이전트 실행은 단일 요청 수명 주기와 연관되어 있지 않습니다.
샌드박스 수명 주기는 독립적으로 최대 절전 모드에 진입하고 재개할 수 있습니다.
모델/제공자 선택과 샌드박스 구현은 별도로 발전할 수 있습니다.
VM은 제어 평면이 되는 대신 일반 실행 환경으로 유지됩니다.
